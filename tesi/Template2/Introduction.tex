	\section {Markov Chains}
	A Discrete time Markov chain is a tuple made up of five element: 
	\begin{itemize}
		\item A set S of States not empty and countable 
		\item A transition probability function that describe the different probability from one state to another. The sum of all transition functions must be one, so the total of probability. 
		\item An initial distribution such that the sum of the initial states is one.
		\item A set of atomic proposition that are distributed throught a labelling function 
	\end{itemize}
	A Markov chain is called finite if the set of States and atomic propositions are finite. If the number of states is infinite the Markov chain is too.

	\section*{What is Prism?}
PRISM is an open-source probabilistic model checker used for the formal verification of stochastic systems.
Unlike traditional testing methods that provide a simple pass/fail answer for specific scenarios, PRISM quantifies system behavior by calculating
the exact probability of whether user-specified properties will occur (e.g., "What is the probability that a network collision occurs within 5 seconds?").
The software automatically computes these probabilities based on the formal model provided as input.
Depending on the nature and time-domain of the system being analyzed, PRISM supports four primary types of models:
  \begin{itemize}
    \item DTMC (Discrete-Time Markov Chain): Used when time evolves in discrete steps or clock cycles
    \item CTMC (Continuous-Time Markov Chain): Used when events can happen at any continuous point in time, with durations modeled using transition rates
    \item MDP (Markov Decision Process): Used to model systems that exhibit both probabilistic behavior and nondeterminism.
      This is ideal for analyzing concurrency, scheduling protocols, or strategic decision-making where a controller interacts with an uncertain environment.
    \item PTA (Probabilistic Timed Automata): Used for systems that require a strict representation of real-time constraints alongside probability, modeled through continuous clocks and timeouts
  \end{itemize}
  
\section{What is GRIP? And how it works?}
Grip is an open-source tool for symmetry reduction in model checking, originally developed at the University of Glasgow.
The tool produces a parsed model of Prism to reduce symmetry in the original model. The process simulate a transaltion of code where the input model modify the different repetition of modules into 
a single one.
